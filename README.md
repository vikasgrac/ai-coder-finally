# FinAlly — AI Trading Workstation

An AI-powered trading terminal for Indian markets (NSE/BSE), built as a capstone for an agentic AI coding course. Streams live prices, simulates a portfolio, and includes an LLM assistant that can analyze positions and execute trades via natural language.

## Features

- **Live price streaming** via SSE — prices flash green/red on tick, with sparkline mini-charts
- **Simulated portfolio** — ₹1,00,000 starting cash, buy/sell with instant market-order fills
- **Portfolio visualization** — treemap heatmap by P&L weight, intraday P&L chart
- **AI chat assistant** — powered by GPT-O3 via Cerebras/OpenRouter; executes trades and manages your watchlist on request
- **10 default NSE tickers**: RELIANCE, TCS, INFY, HDFCBANK, ICICIBANK, HINDUNILVR, SBIN, WIPRO, BAJFINANCE, TATAMOTORS

## Quick Start

```bash
cp .env.example .env
# Add your OPENROUTER_API_KEY to .env
./scripts/start_mac.sh        # macOS/Linux
./scripts/start_windows.ps1   # Windows PowerShell
```

Open [http://localhost:8000](http://localhost:8000).

## Architecture

Single Docker container, single port (8000):

- **Frontend**: Next.js (TypeScript) — static export served by FastAPI
- **Backend**: FastAPI + uv (Python 3.12)
- **Database**: SQLite (volume-mounted at `db/finally.db`)
- **Real-time**: Server-Sent Events (`/api/stream/prices`)
- **AI**: LiteLLM → OpenRouter → Cerebras (`openrouter/openai/gpt-oss-120b`)

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `OPENROUTER_API_KEY` | Yes | API key for LLM chat |
| `TAPETIDE_API_KEY` | No | Live NSE/BSE data; omit to use the built-in simulator |
| `LLM_MOCK` | No | Set `true` for deterministic mock responses (E2E tests) |

## Market Data

By default, prices are generated by a **simulator** using geometric Brownian motion — no external API needed. Set `TAPETIDE_API_KEY` to switch to live NSE/BSE data via Tapetide (~8,200 stocks).

## Stop

```bash
./scripts/stop_mac.sh         # macOS/Linux
./scripts/stop_windows.ps1    # Windows PowerShell
```

Data persists in the `finally-data` Docker volume across restarts.
